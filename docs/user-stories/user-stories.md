# User Stories - Contract Staff Leave Management System

| S.No | User Story | Priority |
|------|-----------|----------|
| 1 | **As a contract staff member**, I want to submit a leave application once through a single interface so that I don't have to fill out multiple forms in different systems.<br>**Acceptance Criteria:** Form captures date range, leave type, reason, parent company reference ID; system validates dates; shows confirmation | HIGH |
| 2 | **As a contract staff member**, I want to select from predefined leave types so that I know whether my leave is chargeable or non-chargeable.<br>**Acceptance Criteria:** Dropdown shows leave types; indicates chargeable/non-chargeable; basic billing impact shown | HIGH |
| 3 | **As a contract staff member**, I want to see the status of my leave applications so that I know if my leave is approved, pending, or rejected.<br>**Acceptance Criteria:** Personal dashboard shows my applications; clear status indicators; view basic details | MEDIUM |
| 4 | **As a hiring manager**, I want to see all leave applications requiring my approval in one dashboard so that I can efficiently manage team leave requests.<br>**Acceptance Criteria:** Dashboard shows pending applications; view staff, leave type, dates; one-click approve/reject | HIGH |
| 5 | **As a hiring manager**, I want to see whether leave applications have been acknowledged by the parent company so that I can follow up on delayed approvals.<br>**Acceptance Criteria:** Shows parent company status; can manually mark as acknowledged; basic time tracking | MEDIUM |
| 6 | **As a finance officer**, I want to automatically calculate monthly billing based on approved leaves so that I can generate accurate invoices without manual calculation.<br>**Acceptance Criteria:** Calculate billable days; apply daily rates; show monthly totals; export CSV | HIGH |
| 7 | **As a finance officer**, I want to generate monthly reconciliation reports comparing both systems so that I can identify and resolve billing discrepancies.<br>**Acceptance Criteria:** Report shows agency vs parent company records; highlight discrepancies; export to CSV | HIGH |
| 8 | **As a finance officer**, I want to track billing variances in real-time so that I can investigate issues before month-end.<br>**Acceptance Criteria:** Dashboard shows total variance; flags large discrepancies (>$500 or >5%); basic investigation workflow | MEDIUM |
| 9 | **As a finance officer**, I want to manage daily billing rates for contract staff so that billing calculations are always current.<br>**Acceptance Criteria:** Simple rate input per staff; used in billing calculations; basic rate management | MEDIUM |
| 10 | **As a compliance officer**, I want to view complete history of all leave-related actions so that I can ensure regulatory compliance.<br>**Acceptance Criteria:** Log shows key actions; search by staff/date; export logs | HIGH |
| 11 | **As an HR manager**, I want to access complete leave records when resolving disputes so that I have all facts.<br>**Acceptance Criteria:** Single view showing application and approval details; basic timeline | MEDIUM |
| 12 | **As an HR manager**, I want to analyze historical leave patterns so that I can identify trends and optimize leave policies.<br>**Acceptance Criteria:** Basic reports showing leave counts by type/month | LOW |
| 13 | **As a system administrator**, I want to assign user roles so that users have appropriate access.<br>**Acceptance Criteria:** Basic user accounts with simple role assignment; login for different dashboards | HIGH |
| 14 | **As a system administrator**, I want to configure email notifications so that users receive important updates.<br>**Acceptance Criteria:** Basic email templates; send on submission/approval/rejection | HIGH |
| 15 | **As a system administrator**, I want to configure parent company settings so that the system can reference different companies.<br>**Acceptance Criteria:** Add parent company names; basic email settings | MEDIUM |
| 16 | **As a system**, I want to send automated email notifications when leave status changes so that relevant parties are informed.<br>**Acceptance Criteria:** Send email on submission (to manager); send email on approval/rejection (to staff) | HIGH |
| 17 | **As a system**, I want to validate leave applications automatically so that I prevent errors.<br>**Acceptance Criteria:** Validate dates (no past dates); check for overlaps; basic validation rules | HIGH |
| 18 | **As a system**, I want to automatically calculate billing each month so that finance reports are always ready.<br>**Acceptance Criteria:** Auto-calculate on 1st of month; generate reconciliation report; flag variances | HIGH |
